# VerifyBot

Discord-бот верификации участников по кнопке с анкетой и модерацией заявок.

## Поток работы

1. Админ пишет `/panel` в нужном канале — бот публикует панель с кнопками
   **Верифицироваться** и **Проверить аккаунт**.
2. Пользователь нажимает **Верифицироваться** и заполняет анкету из 4 полей:
   - Имя
   - Возраст
   - Согласие носить тег (Да/Нет)
   - Как узнали про нас?
3. Заявка отправляется в канал заявок с пингом роли саппорта и кнопками
   **Принять / Отклонить** (доступны только роли саппорта или администраторам).
4. **Принять** → пользователю выдаётся роль верификации, ему приходит ЛС об одобрении.
5. **Отклонить** → открывается модалка с причиной → причина отправляется пользователю в ЛС.

## Установка и запуск

```bash
cd verifybot
npm install
npm start
```

## Настройка (`.env`)

| Переменная | Назначение |
|---|---|
| `DISCORD_TOKEN` | Токен бота |
| `APPLICATIONS_CHANNEL_ID` | Канал, куда приходят заявки |
| `SUPPORT_ROLE_ID` | Роль, которая рассматривает заявки |
| `VERIFY_ROLE_ID` | Роль, выдаваемая при принятии |
| `GUILD_ID` | (опц.) ID сервера для мгновенной регистрации `/panel` |

## Важно перед запуском

- В **Discord Developer Portal → Bot** включите **Server Members Intent** и
  **Presence Intent** (без них бот не запустится / не определит устройство).
- Роль бота должна быть **выше** выдаваемой роли (`VERIFY_ROLE_ID`) в списке ролей,
  иначе выдача роли не сработает.
- **Кастомные эмодзи и картинки** в `src/config.js` / `src/components.js`
  (статус-иконки `<:yellow:...>` и ссылка на изображение панели) принадлежат другому
  серверу и могут не отображаться. Замените их на эмодзи/ссылки своего сервера.
- ЛС-уведомления не дойдут, если у пользователя закрыты личные сообщения — это не ошибка.
